post https://restapi.ordergroove.com/products/{product\_id}/selection\_rules/time\_window/manage/

This API allows you to take an existing standard product and configure it into a valid time-window rotating product or to manage an existing configuration.

## 

Time-Window selection rule validations

[](#time-window-selection-rule-validations)

There are **4 validation rules** for defining Time-Window selection rules:

1.  You must define at least one selection rule (we wouldnâ€™t have anything to choose otherwise)
2.  One of the selection rules must have a starting date in the past (so that we donâ€™t ever have an invalid state where we donâ€™t know which product should be delivered)
3.  No two selection rules can have the same starting date (otherwise we wouldnâ€™t know which product to choose between them, but you can have multiple selection rules with the same product)
4.  All starting dates must be in ISO8601 format with timezone. The timezone will be stored in UTC.

If validation fails no changes will be made to the product and no selection rules will be created, updated or deleted.

  

## 

How to manage a Time-Window based Rotation using this API

[](#how-to-manage-a-time-window-based-rotation-using-this-api)

1.  Choose an existing Standard or Rotating Product, if you donâ€™t have one yet you will need to create one before you proceed with this configuration.
2.  Decide which **selection rules** you want to add, update or delete:
    1.  All the rule validation explained above apply here, meaning that if you donâ€™t follow them your update won't happen;
    2.  Your changes will be reflected in all order items that havenâ€™t had their Order Reminder sent or gone through the Send Now flow;
    3.  You can add a new selection rule;
    4.  You can delete an existing selection rule;
    5.  You can update an existing selection rule with a different product and or starting date;
3.  Hit our API with your configuration, and your Time-Window based Rotating Product setup is complete!
